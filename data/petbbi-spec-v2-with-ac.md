## 1. 반려동물 프로필 등록

반려동물의 기본 정보와 대표 사진을 등록하여 AI 분석의 기반 데이터를 생성하는 기능.

### 1-1. 기본 정보 입력

- 입력 항목: 이름(필수), 종류(필수), 품종(선택), 생일(선택), 성별(필수)
- 이름: 1자 이상 20자 이하, 이모지 입력 허용
- 생일: 캘린더 선택, 오늘 이전 날짜만 선택 가능
- 계정당 반려동물 프로필 최대 **5마리**
- 입력 완료 후 "다음" 버튼 → 대표 사진 업로드 화면으로 이동

### 1-2. 대표 사진 업로드

- 카메라 촬영 또는 갤러리에서 선택, 1장 등록
- 허용 형식: JPG, PNG, HEIC
- 파일 용량 상한: **10MB** 초과 시 업로드 차단
- 최소 해상도: **512×512px** 미만 시 경고 팝업 노출 → 사용자가 확인/취소 선택
- 업로드 후 미리보기 화면에서 확인 → "등록" 버튼으로 확정

### 1-3. 프로필 수정·삭제

- 기본 정보 및 대표 사진 언제든 수정 가능
- 프로필 삭제 시 연결된 캐릭터·일기 모두 삭제
- 프로필 목록에서 대표 반려동물 1마리 지정 가능

### ✅ 수용 기준 (Acceptance Criteria)

**AC-1. 정상 등록**
- Given: 이름/종류/성별 입력 완료, 10MB 이하 유효 이미지 업로드
- When: 등록 버튼 클릭
- Then: 프로필 목록에 신규 카드 추가, 프로필 화면 상단에 대표 사진 및 기본 정보 표시

**AC-2. 필수값 누락**
- Given: 이름 또는 종류 미입력
- When: 다음 버튼 클릭
- Then: 미입력 필드에 빨간 테두리 + "필수 입력 항목입니다" 인라인 오류 메시지 노출, 다음 화면 미이동

**AC-3. 5마리 한도 초과**
- Given: 반려동물 5마리 등록 완료 상태
- When: 6번째 등록 시도
- Then: 등록 버튼 비활성화(그레이아웃) + "최대 5마리까지 등록 가능합니다" 팝업 노출

**AC-4. OS 권한 미허용**
- Given: OS 카메라·갤러리 권한 거부 상태
- When: 대표 사진 업로드 버튼 클릭
- Then: "사진 접근 권한이 필요합니다. 설정에서 허용해 주세요" 팝업 + [설정으로 이동] 버튼 제공, 파일 선택창 미노출

**AC-5. 저해상도 이미지**
- Given: 512×512px 미만 이미지 선택
- When: 업로드 시도
- Then: "512×512px 미만 이미지입니다. 업로드하시겠습니까?" 경고 팝업 → [확인] 시 허용, [취소] 시 파일 선택 화면 유지

## 2. AI 캐릭터 생성

업로드된 대표 사진을 AI가 분석하여 외형·성격 특징이 반영된 전용 캐릭터를 생성하는 기능.

### 2-1. 캐릭터 생성 요청

- 프로필 화면에서 "AI 캐릭터 만들기" 버튼 클릭 → 생성 시작
- AI가 대표 사진의 외형(털, 귀 모양, 체형 등)과 사용자가 선택한 성격 키워드(최대 3개)를 분석
- 생성 중 로딩 화면에 진행률(%)과 예상 완료 시간 표시
- 평균 생성 시간: 15~30초, 타임아웃 기준: **60초**

### 2-2. 생성 결과 표시

- 생성 완료 시 정면·측면·정수리 3종 캐릭터 이미지를 카드 형태로 표시
- AI가 분석한 성격 요약 문구 함께 표시 (예: "호기심이 많은 모험가")
- "내 캐릭터로 저장" 버튼으로 확정
- 저장된 캐릭터는 프로필·일기·커뮤니티 게시물에 사용

### 2-3. 재생성 정책

- "다시 생성" 버튼으로 재생성 가능
- 재생성 한도: **1일 3회**
- 초기화 시점: **KST 기준 자정(00:00)**
- 잔여 횟수 버튼 하단에 항상 표시 (예: 2/3)
- 재생성 시 이전 결과와 다른 변화형 캐릭터 생성

### ✅ 수용 기준 (Acceptance Criteria)

**AC-1. 정상 생성**
- Given: 선명한 대표 사진 등록, 성격 키워드 최소 1개 선택
- When: AI 캐릭터 만들기 버튼 클릭
- Then: 60초 내 3종 캐릭터 이미지 및 성격 요약 문구 카드 표시, 저장 버튼 활성화

**AC-2. 저품질 이미지로 생성 시도**
- Given: 반려동물 얼굴이 가려지거나 흐린 이미지 등록
- When: 캐릭터 생성 요청
- Then: "사진을 분석할 수 없습니다. 반려동물의 얼굴이 잘 보이는 사진으로 다시 시도해 주세요" 오류 팝업, 앱 크래시 없음

**AC-3. 생성 중 네트워크 차단**
- Given: 캐릭터 생성 로딩 중
- When: 네트워크 연결 차단
- Then: "네트워크 연결이 끊겼습니다" 배너 + "다시 시도" 버튼 노출, 해당 시도 재생성 횟수 미차감

**AC-4. 재생성 한도 초과**
- Given: 당일 재생성 3회 소진 완료
- When: "다시 생성" 버튼 클릭 시도
- Then: 버튼 비활성화(그레이아웃) + "오늘 재생성 횟수를 모두 사용했습니다 (0/3)" 문구 노출

**AC-5. AI 서버 완전 실패**
- Given: AI 서버 오류 또는 60초 타임아웃 발생
- When: 생성 요청
- Then: "캐릭터 생성에 실패했습니다. 잠시 후 다시 시도해 주세요" 팝업, 재생성 횟수 미차감, 앱 크래시 없음

## 3. AI 일기 자동작성

생성된 캐릭터가 반려동물의 하루를 1인칭 시점으로 자동 작성하는 기능.

### 3-1. 일기 생성 트리거

- 사용자가 사진 업로드 또는 활동 기록 시 일기 생성 트리거
- 자동 생성: **KST 기준 오후 9시** (기기 로컬 시간대 무관, 서버 시간 기준)
- 수동 생성: "오늘의 일기 쓰기" 버튼 클릭
- 생성 전 사용자가 오늘의 감정 키워드(기쁨/슬픔/신남/피곤 등) 선택 — **필수**
- 선행 조건: 해당 반려동물의 캐릭터 생성 완료 상태

### 3-2. 일기 내용 생성

- 캐릭터 성격과 선택된 키워드 반영, 1인칭 일기 텍스트 자동 생성
- 일기 길이: **200~400자**
- 일기에 캐릭터 이미지와 날짜 함께 표시
- 생성된 일기 자동 저장, "일기장" 탭에서 날짜별 조회 가능
- 사용자 직접 수정·삭제 가능

### 3-3. 다국어 일기

- 사용자 설정 언어로 생성 (한국어/영어/일본어)
- 언어 변경 시 이후 생성되는 일기부터 적용, 기존 일기 소급 변경 없음

**AC-4. 해외 시간대 기기에서 자동 생성**
- Given: 기기 시간대 UTC+0 설정
- When: 서버 시간 KST 오후 9시 도달
- Then: 기기 로컬 시간과 무관하게 KST 기준으로 자동 일기 생성 트리거

**AC-5. 앱 강제 종료 후 재진입**
- Given: 일기 생성 진행 중 앱 강제 종료
- When: 앱 재진입
- Then: 생성 완료된 경우 일기장 자동 저장 / 미완료 시 "일기 생성이 완료되지 않았습니다. 다시 생성하시겠습니까?" 팝업 노출

## 4. 커뮤니티 피드

사용자들이 캐릭터·일기 기반 콘텐츠를 공유하고 소통하는 기능.

### 4-1. 게시물 작성

- 캐릭터 이미지 또는 일기 첨부 가능
- 텍스트 본문: 최대 **500자**
- 이미지 첨부: 최대 **4장**
- 해시태그: 최대 **10개**
- "게시" 버튼 클릭 시 피드에 즉시 노출

### 4-2. 피드 조회

- 최신순/인기순 정렬 선택 가능
- 게시물 좋아요·댓글 작성 가능
- 무한 스크롤: **20개 단위** 로드
- 다른 사용자 프로필 클릭 시 해당 사용자 게시물 모아보기
- 첨부된 캐릭터·일기의 원본 삭제 시: 해당 영역에 "삭제된 콘텐츠입니다" 플레이스홀더 표시

### 4-3. 검색 및 팔로우

- 해시태그 또는 닉네임으로 검색
- 팔로우 시 해당 사용자 신규 게시물 피드 우선 노출
- 검색 결과: 게시물 탭 / 사용자 탭 구분 표시

---

### 📺 화면 상태 정의

| 상태 | 화면 표시 |
|------|---------|
| **정상** | 게시물 카드 목록 (캐릭터 이미지·텍스트·해시태그·좋아요·댓글 버튼) |
| **로딩** | 스켈레톤 UI (카드 형태 플레이스홀더) |
| **에러** | 로드 실패: "게시물을 불러올 수 없습니다" + [다시 시도] 버튼 / 게시 실패: "게시에 실패했습니다. 다시 시도해 주세요" 토스트 3초 |
| **빈 상태** | 팔로우 0명: 빈 상태 일러스트 + "아직 게시물이 없어요" 문구 + "다른 반려인 찾기" CTA 버튼 |

---

### 📐 수치·정책 기준

| 항목 | 기준값 |
|------|--------|
| 본문 최대 글자 수 | 500자 (초과 시 입력 차단) |
| 이미지 첨부 최대 수 | 4장 |
| 해시태그 최대 수 | 10개 (초과 시 입력 필드 비활성화) |
| 무한 스크롤 단위 | 20개 |
| 첨부 원본 삭제 시 | "삭제된 콘텐츠입니다" 회색 플레이스홀더 표시, 나머지 게시물 정상 노출 |
| 빈 게시물 방지 | 본문·첨부 모두 없을 시 게시 버튼 비활성화 |

---

### ✅ 수용 기준 (Acceptance Criteria)

**AC-1. 정상 게시**
- Given: 본문 또는 첨부 파일 1개 이상 입력
- When: 게시 버튼 클릭
- Then: 피드에 게시물 즉시 노출, 해시태그 링크 활성화, 좋아요·댓글 버튼 표시

**AC-2. 빈 게시물 방지**
- Given: 본문과 첨부 파일 모두 없음
- When: 게시 버튼 클릭
- Then: 게시 버튼 비활성화(그레이아웃) 유지, 클릭 불가

**AC-3. 해시태그 한도 초과**
- Given: 해시태그 10개 입력 완료
- When: 11번째 해시태그 입력 시도
- Then: 입력 필드 비활성화 + "최대 10개까지 입력 가능합니다" 토스트 메시지 노출

**AC-4. 본문 글자 수 초과**
- Given: 본문 500자 입력 완료
- When: 501번째 글자 입력 시도
- Then: 입력 차단, 잔여 글자 수 0 표시

**AC-5. 첨부 원본 삭제 후 조회**
- Given: 캐릭터 또는 일기를 첨부한 게시물 존재, 이후 원본 삭제
- When: 해당 게시물 조회
- Then: 삭제된 원본 영역에 "삭제된 콘텐츠입니다" 회색 플레이스홀더 표시, 앱 크래시 없음

## 5. 알림

서비스 이벤트와 커뮤니티 활동을 사용자에게 전달하는 기능.

### 5-1. 알림 종류

| 알림 유형 | 분류 |
|---------|------|
| 일기 생성 완료 | 선택적 |
| 캐릭터 생성 완료 | 선택적 |
| 좋아요 | 선택적 |
| 댓글 | 선택적 |
| 팔로우 | 선택적 |
| 서비스 공지 (일반) | 선택적 |
| 서비스 긴급 공지 | **필수** — 전체 알림 OFF 시에도 강제 발송 |

### 5-2. 알림 설정

- 알림 유형별 ON/OFF 개별 설정 가능
- 푸시 알림 / 이메일 알림 채널 선택 가능
- 설정 변경 즉시 반영
- 긴급 공지: 앱 내 상단 고정 배너 + 등록 이메일로 별도 발송 (설정 무관)

---

### 📺 화면 상태 정의

| 상태 | 화면 표시 |
|------|---------|
| **정상** | 알림 유형별 내역 리스트, 읽음/안읽음 표시 |
| **로딩** | 알림 설정 저장 중 스피너 |
| **에러** | OS 권한 거부: "알림 수신을 위해 기기 설정에서 알림 권한을 허용해 주세요" 팝업 + [설정으로 이동] 버튼 |
| **빈 상태** | "아직 알림이 없어요" 문구 |

---

### 📐 수치·정책 기준

| 항목 | 기준값 |
|------|--------|
| 선택적 알림 | 사용자 설정에 따라 수신 여부 결정 |
| 긴급 공지 | 전체 알림 OFF 시에도 앱 내 배너 + 이메일 강제 발송 |
| OS 권한 거부 + 앱 내 ON 설정 시 | OS 권한 요청 팝업 노출, 권한 허용 전까지 실제 알림 미수신 |
| 설정 변경 반영 시점 | 즉시 |

---

### ✅ 수용 기준 (Acceptance Criteria)

**AC-1. 정상 알림 수신**
- Given: 좋아요 알림 ON, OS 푸시 권한 허용
- When: 다른 사용자가 내 게시물에 좋아요 클릭
- Then: 기기 잠금 화면 및 알림 센터에 푸시 즉시 수신, 앱 내 알림 탭에 내역 기록

**AC-2. 알림 설정 즉시 반영**
- Given: 일기 생성 알림 ON 상태에서 OFF로 변경
- When: 일기 자동 생성 완료
- Then: 해당 유형 알림 미수신, 알림 탭 내역 미기록

**AC-3. OS 권한 거부 상태**
- Given: OS 알림 권한 거부 상태
- When: 앱 내 좋아요 알림 ON으로 변경 시도
- Then: "알림 수신을 위해 기기 설정에서 알림 권한을 허용해 주세요" 팝업 + [설정으로 이동] 버튼 제공

**AC-4. 긴급 공지 강제 발송**
- Given: 사용자 전체 알림 OFF 설정
- When: 서비스 긴급 공지 발송
- Then: 앱 내 상단 고정 배너 노출 + 등록 이메일로 별도 발송

## 6. 계정 및 인증

사용자 계정 생성·관리·삭제 및 인증을 담당하는 기능.

### 6-1. 회원가입

- 이메일 가입: 이메일, 비밀번호(8자 이상), 닉네임(2~15자) 입력
- 소셜 로그인: 구글 / 애플 / 카카오 지원
- 가입 완료 후 이메일 인증 메일 발송 → **24시간 이내** 인증 완료 필수
- 인증 링크 만료 시: "인증 링크가 만료되었습니다" 안내 + [인증 메일 재발송] 버튼 제공
- 만 14세 이상만 가입 가능 (생년월일 확인)

### 6-2. 소셜 계정 충돌 정책

- 이메일 계정과 동일 이메일의 소셜 로그인 시도 시: "해당 이메일은 이미 이메일 계정으로 가입되어 있습니다. 이메일 로그인을 이용해 주세요" 안내 팝업, 소셜 로그인 차단
- 소셜 계정과 동일 이메일의 이메일 가입 시도: "이미 소셜 로그인으로 가입된 이메일입니다" 안내

### 6-3. 프로필 관리

- 닉네임(2~15자), 프로필 사진, 자기소개 수정 가능
- 비밀번호 변경: 현재 비밀번호 확인 후 신규 입력

### 6-4. 계정 삭제

- 설정 → "계정 삭제" → 삭제 사유 선택 → 비밀번호 재확인 → 최종 삭제
- 삭제 시 등록된 반려동물 프로필·캐릭터·일기·게시물 모두 삭제
- 삭제 후 **30일간** 데이터 보관 → 30일 내 재로그인 시 복구 가능
- 삭제 도중 강제 종료 후 재진입 시: 삭제 미완료면 정상 로그인 유지 / 완료면 "계정이 삭제되었습니다" 안내 후 로그인 화면 이동, 데이터 정합성 보장

### 6-5. 삭제 후 재가입

- 삭제 후 30일 이내 동일 이메일로 재가입 시도: "이전에 사용하던 계정이 있습니다. 계정을 복구하시겠습니까?" 팝업 → [복구하기] / [새 계정 만들기] 선택 제공

---

### 📺 화면 상태 정의

| 상태 | 화면 표시 |
|------|---------|
| **정상** | 로그인 완료 → 홈 화면 이동 |
| **로딩** | 로그인·가입 처리 중 버튼 로딩 스피너 |
| **에러** | 이메일 중복: "이미 사용 중인 이메일입니다" 인라인 오류 / 인증 만료: "인증 링크가 만료되었습니다" 안내 + 재발송 버튼 / 소셜 충돌: "이미 이메일 계정으로 가입된 이메일입니다" 팝업 / 비밀번호 불일치: "비밀번호가 올바르지 않습니다" 인라인 오류 |
| **빈 상태** | 해당 없음 (인증 플로우 특성상 빈 상태 없음) |

---

### 📐 수치·정책 기준

| 항목 | 기준값 |
|------|--------|
| 비밀번호 최소 길이 | 8자 이상 |
| 닉네임 글자 수 | 2~15자 |
| 이메일 인증 유효 시간 | 24시간 |
| 인증 만료 시 처리 | 만료 안내 + 재발송 버튼 제공 |
| 최소 가입 연령 | 만 14세 이상 |
| 계정 삭제 후 데이터 보관 기간 | 30일 |
| 소셜 계정 충돌 시 | 기존 이메일 계정 유지, 소셜 로그인 차단 + 안내 팝업 |
| 삭제 프로세스 원자성 | 강제 종료 후 재진입 시 상태 일관성 보장, 데이터 정합성 손상 없음 |

---

### ✅ 수용 기준 (Acceptance Criteria)

**AC-1. 정상 가입 및 인증**
- Given: 사용되지 않은 이메일, 8자 이상 비밀번호, 2~15자 닉네임
- When: 가입 완료 후 24시간 내 인증 링크 클릭
- Then: "이메일 인증이 완료되었습니다" 토스트 노출, 앱 내 로그인 상태로 자동 전환

**AC-2. 중복 이메일 가입 시도**
- Given: 이미 가입된 이메일 주소
- When: 신규 가입 시도
- Then: 이메일 필드 하단에 "이미 사용 중인 이메일입니다" 인라인 오류 + "로그인하기" 링크 노출, 가입 진행 불가

**AC-3. 인증 링크 만료**
- Given: 인증 메일 발송 후 24시간 초과
- When: 만료된 링크 클릭
- Then: "인증 링크가 만료되었습니다" 안내 화면 + [인증 메일 재발송] 버튼 제공

**AC-4. 소셜 계정 충돌**
- Given: 이메일 계정으로 가입된 이메일과 동일한 이메일로 소셜 로그인 시도
- When: 소셜 로그인 버튼 클릭
- Then: "해당 이메일은 이미 이메일 계정으로 가입되어 있습니다. 이메일 로그인을 이용해 주세요" 안내 팝업, 소셜 로그인 차단

**AC-5. 삭제 후 30일 이내 재가입**
- Given: 계정 삭제 후 30일 이내, 동일 이메일로 재가입 시도
- When: 회원가입 진행
- Then: "이전에 사용하던 계정이 있습니다. 계정을 복구하시겠습니까?" 팝업 → [복구하기] 클릭 시 기존 데이터 복원 / [새 계정 만들기] 클릭 시 신규 가입 진행

**AC-6. 계정 삭제 강제 종료 후 재진입**
- Given: 계정 삭제 비밀번호 재확인 완료 후 최종 처리 단계에서 앱 강제 종료
- When: 앱 재진입
- Then: 삭제 미완료 시 정상 로그인 유지 / 삭제 완료 시 "계정이 삭제되었습니다" 안내 후 로그인 화면 이동, 데이터 정합성 손상 없음
